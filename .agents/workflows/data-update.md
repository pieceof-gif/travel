# 데이터 최신화 워크플로우

odiga의 신뢰도는 데이터 정확도에 달려있습니다.
아래 항목을 주기적으로 점검하고 업데이트합니다.

---

## AI 자동 점검 규칙 (세션 시작 시)

odiga 관련 대화가 시작될 때마다 AI는 다음을 먼저 확인합니다:

1. 마지막 데이터 업데이트 날짜 (`현재 데이터 현황` 섹션)
2. 마지막 업데이트로부터 **분기(3개월)** 이상 지났으면 점검 알림
3. 주요 여행지 비자·입국 정책 변경 웹 검색
4. 이슈 발견 시 작업 전 먼저 사용자에게 보고 후 투두 작성

**점검 대상 여행지:**
다낭, 방콕, 발리, 오사카, 도쿄, 싱가포르, 세부, 타이페이, 홍콩, 마카오, 베이징, 상하이, 괌, 사이판, 파리, 바르셀로나, 리스본, 시드니, 몰디브, 쿠알라룸푸르, 치앙마이, 후쿠오카, 삿포로, 오키나와, 푸켓, 푸꾸옥, 호치민, 하노이, 나트랑, 보라카이

## 점검 주기 가이드

| 항목 | 권장 주기 |
|---|---|
| 항공 기준 요금 (`baseAir`) | 분기 1회 |
| 숙박 기준 요금 (`baseHotel`) | 분기 1회 |
| 현지 일일 지출 (`daily`) | 반기 1회 |
| 필수 앱 목록 (`DEST_APPS`) | 반기 1회 |
| 출입국 비자 정보 (`DEST_ENTRY`) | 분기 1회 (비자 정책 변경 시 즉시) |
| 환전 팁 / 은행 위치 (`DEST_FX_TIPS`) | 분기 1회 |
| 음식 추천 (`food`) | 반기 1회 |
| 체험 추천 (`experiences`) | 반기 1회 |
| 여행지 설명 narratives | 연 1회 |

## 콘텐츠 정확도 점검 (하드코딩 데이터)

요금 외에 **내용 자체가 달라졌는지** 확인합니다.

### 음식 추천 (`food`)

각 여행지 데이터의 `food` 배열 안 식당/음식 목록 점검:
- 식당이 폐업했는지 (구글맵 검색으로 빠르게 확인)
- 유행이 지나 더 이상 대표 음식이 아닌 항목
- 새로 유명해진 현지 음식 또는 식당

**확인 방법:** 구글맵 리뷰 + 최근 여행 블로그 검색

---

### 체험 추천 (`experiences`)

각 여행지의 `experiences` 또는 관광 항목 점검:
- 운영 중단된 체험 / 폐장된 관광지
- 예약 방식 변경 (현장 → 사전 예약 필수 등)
- 새로 생긴 인기 체험

**확인 방법:** Klook, 트립어드바이저 최신 리뷰 확인

---

### 환전 / 은행 위치 (`DEST_FX_TIPS`)

`DEST_FX_TIPS` 매핑의 환전 팁 점검:
- 현지 추천 환전소 또는 은행 위치 변경 여부
- 수수료 정책 변경 (예: 특정 앱 환전 수수료 무료 → 유료)
- 트래블월렛, 하나머니 등 환전 앱 정책 변경

**확인 방법:** 각 환전 앱 공지사항 + 여행 커뮤니티(네이버 카페)

---

### 출입국 / 비자 (`DEST_ENTRY`)

`DEST_ENTRY` 매핑의 입국 조건 점검:
- 무비자 → 비자 필요로 변경 (또는 반대)
- 전자여행허가(ETA) 신규 도입
- 입국 시 필요 서류 변경

**확인 방법:** 외교부 해외안전여행 (www.0404.go.kr) + 각국 대사관 공지

---

### 즉시 업데이트 기준

아래 상황은 발견 즉시 반영합니다:
- 비자 정책 변경 (잘못된 정보 → 사용자 입국 거부 위험)
- 식당/관광지 폐업 (신뢰도 하락)
- 환전 앱 서비스 종료

---

## 1단계: 항공·숙박 요금 확인

각 여행지의 `baseAir`, `baseHotel` 값을 실제 시장가와 비교합니다.

**확인 방법:**
- 네이버 항공 또는 스카이스캐너에서 인천 출발 기준 최저가 검색
- Agoda / Booking.com에서 숙박 평균가 확인
- 20% 이상 차이 나면 업데이트

**업데이트 위치:** `index.html` > `v1_0_9_DEST_DATA` 배열 내 각 여행지 객체

---

## 2단계: 필수 앱 목록 확인

`DEST_APPS` 매핑에서 각 나라별 앱이 여전히 유효한지 확인합니다.

체크 항목:
- 앱이 서비스 종료되지 않았는지
- 앱 이름/아이콘 URL이 유효한지
- 새로 유행하는 필수 앱이 생겼는지

---

## 3단계: 비자/출입국 정보 확인

`DEST_ENTRY` 또는 연동된 비자 데이터에서 확인:
- 무비자 입국 가능 여부 변경
- 도착 비자 정책 변경
- 전자여행허가(ETA) 신규 도입 등

**빠른 확인:** 외교부 해외안전여행 사이트 (www.0404.go.kr)

---

## 4단계: 업데이트 커밋

```bash
git add -A && git commit -m "data: 여행 데이터 최신화 — YYYY-MM"
```

---

## 5단계: 즉시 업데이트가 필요한 경우

아래 상황은 즉시 반영합니다:
- 특정 국가 비자 정책 변경 (사용자에게 잘못된 정보 제공 위험)
- 항공/숙박 요금이 데이터 대비 2배 이상 차이 나는 경우
- 필수 앱 서비스 종료
